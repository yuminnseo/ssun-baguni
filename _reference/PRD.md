# PRD: 쓴바구니

> **문서 상태**: 작업 중 (PRD 단계)
> **최종 업데이트**: 2026-06-08
> **참고**: Home UI는 추후 제공되는 프로토타입 파일을 최종 기준으로 삼는다. 현재 PRD에 기술된 홈 화면 구조는 요구사항 기준이며, 세부 UI는 프로토타입 확정 후 갱신한다.

---

## 1. 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | 쓴바구니 |
| 한 줄 설명 | 구매한 물건을 누끼 사진으로 장바구니에 모아 한 눈에 보여주며, 재정관리 초보자가 자신의 소비 기록을 즐겁게 인식하고 기록 습관을 만들 수 있게 돕는 시각형 가계부 앱 |
| 타겟 유저 | 매일 가계부를 쓰려고 해도 금방 포기하는 재정 관리 초보자 (주 타겟: 20대 여성) |
| 기술 스택 | Next.js (App Router), Tailwind CSS |
| 디자인 톤 | 키치(Kitschy) + 미니멀(Minimal) |

---

## 2. 문제 정의

### 2.1 사용자 페인포인트

- 기존 가계부 앱은 숫자 입력 중심이라 지루하고 지속하기 어렵다.
- 소비를 기록하는 행위 자체가 의무감·죄책감으로 느껴져 쉽게 포기한다.
- 20대 재정 관리 초보자에게 적합한, 진입 장벽이 낮은 도구가 부족하다.

### 2.2 해결 방향

- 사진 찍기라는 직관적이고 즐거운 행동으로 소비 기록 진입 장벽을 낮춘다.
- 장바구니·영수증 등 친숙한 쇼핑 은유를 활용해 소비 인식을 긍정적으로 전환한다.
- 챌린지와 친구 공유 기능으로 기록 습관 유지를 유도한다.

## 3. 정보 구조 (IA)

```
쓴바구니
├── 홈
│   ├── 장바구니 탭 (날짜별, 가로 스와이프 또는 날짜 선택)
│   │   └── 누끼 이미지 배치 + 가격 태그
│   │       ├── 물건 클릭 → 물건 카드 (편집/삭제)
│   │       └── 빈 상태 → CTA (무지출 DAY / 구매한 물건 담기)
│   └── 영수증 탭 (오늘 소비 목록, 감열지 UI)
├── 캘린더
│   ├── 세그먼트: 월간 / 주간
│   ├── 캘린더 셀 (날짜 + 누끼 썸네일)
│   │   └── 셀 클릭 → 홈 해당 날짜로 이동
│   ├── 하단 장바구니 카트 (기간/날짜 물품 모아보기, 인터랙션)
│   └── 분석 영역 (최다 소비 품목, 가장 비싼 물건, 소비 이유 통계, 무지출일)
├── 친구
│   ├── 친구 목록 및 그룹 관리
│   ├── 친구 장바구니 구경
│   └── 공유 범위 / 금액 노출 설정
└── MY
    ├── 챌린지 현황
    ├── 장바구니 테마 / 꾸미기 관리
    └── 계정 관리
```

| 탭 | 아이콘 방향 | 설명 |
|---|---|---|

| 캘린더 | — | 주간/월간 소비 내역 및 분석 |

## 5. 홈 화면

> **참고**: 홈 화면의 세부 UI는 추후 제공될 프로토타입 파일을 최종 기준으로 한다. 아래는 구조 및 요구사항 기준이다.

### 5.1 화면 구조

```
┌──────────────────────────────┐
│ [헤더]                        │
│  날짜 (예: 2026.06.08)  [컬러] │
├──────────────────────────────┤
│ [세그먼트 컨트롤]              │
│   장바구니   │   영수증         │
├──────────────────────────────┤
│                              │
│  [중앙 영역]                  │
│  장바구니 또는 영수증 노출       │
│  (기본값: 장바구니)            │
│                              │
├──────────────────────────────┤
│ [하단]                        │
│  Navigation   [Plus 버튼]     │
└──────────────────────────────┘
```

### 5.2 헤더

- **날짜 표시**: `YYYY.MM.DD` 형식 (예: 2026.06.08)
- **컬러 아이콘**: 오늘의 장바구니 테마 컬러를 나타내는 아이콘. 클릭 시 컬러 변경 페이지로 이동.

### 5.3 세그먼트 컨트롤

- **장바구니 탭**: 날짜별 누끼 이미지 장바구니 뷰
- **영수증 탭**: 오늘 소비 목록을 감열지 스타일로 표시

### 5.4 장바구니 탭 요구사항

- 날짜별 장바구니 안에 누끼 이미지가 배치된다.
- 누끼 이미지에는 **가격 태그**가 부착된다.
  - 1px 짧은 블랙 실선 + 작은 태그에 가격 표기 (예: ₩10,000)
  - 태그 클릭·드래그 시 실선이 늘어나는 인터랙션 효과
- 누끼 이미지에는 **Offset Path 4px** (`#111111`) 처리로 스티커 느낌 부여
- 장바구니 안의 물건은 **드래그로 위치 이동 가능** (구체적인 움직임은 추후 프로토타입 참고)
- 날짜 간 이동은 헤더 날짜 클릭 → 날짜 선택 바텀시트 방식으로 처리

### 5.5 영수증 탭 요구사항

- 오늘 소비 합계와 품목 리스트를 감열지 스타일 UI로 표시
- 영수증 컬러는 해당 날짜의 장바구니 테마 컬러에 대응
- 점선 구분선, 모노스페이스 폰트 사용

### 5.6 빈 장바구니 상태

장바구니에 물건이 없을 때는 CTA 버튼 2개를 제공한다.

| 버튼 | 동작 |
|---|---|
| 무지출 DAY | 해당 날짜를 무지출로 기록 |
| 구매한 물건 담기 | 물건 추가 플로우 진입 (Plus 버튼과 동일) |

### 5.7 홈 화면 인터랙션 요구사항

| 인터랙션 | 동작 |
|---|---|
| 날짜 클릭 | 날짜 선택 바텀시트 등장 |
| Plus 버튼 클릭 | 카메라 / 갤러리 선택 바텀시트 등장 |
| 컬러 아이콘 클릭 | 컬러 변경 페이지로 이동 (UI 미정, 요구사항만 반영) |
| 장바구니 물건 드래그 | 장바구니 내 위치 이동 (세부 모션은 프로토타입 기준) |
| 물건 클릭 | 물건 상세 카드 등장 (삭제 / 편집 가능, UI 미정) |
| 오늘이 아닌 날짜 조회 | 화면에 '오늘' chip 노출 → 클릭 시 오늘로 복귀 (UI 미정) |

### 5.8 컬러 변경 페이지 (요구사항)

> UI 미제공. 아래 요구사항만 정리.

- 날짜별로 장바구니 테마 컬러를 개별 설정할 수 있어야 한다.
- 파스텔 계열 프리셋 컬러 팔레트 제공
- 챌린지 보상으로 잠금 해제되는 추가 컬러가 있을 수 있다.

### 5.9 물건 카드 (요구사항)

> UI 미제공. 아래 요구사항만 정리.

- 물건 클릭 시 화면 중앙에 카드 형태로 노출
- 반투명 블러 배경 처리
- 카드 내 기능: 품목명 편집, 가격 편집, 소비 이유 편집, 삭제
- 카드 간 스와이프로 이전/다음 품목 이동
- 배경 영역 탭으로 카드 닫기

## 6. 물건 추가 플로우

Plus 버튼 또는 빈 장바구니 CTA(구매한 물건 담기)를 통해 진입한다.

```
1. Plus 버튼 클릭
      │
2. 바텀시트: 카메라 / 갤러리 선택
      │
3. 사진 촬영 또는 이미지 선택
      │
4. AI 누끼 처리 (자동)
      │
5. 카테고리 선택
   - 미리 정의된 카테고리 리스트에서 선택
   - AI가 이미지 분석 기반으로 카테고리 추천
   - 사용자가 직접 선택하거나 변경 가능
      │
6. 가격 입력
      │
7. 소비 이유 선택
   - 예시 리스트: 귀여워서 / 필요해서 / 배고파서 / 할인이라서 / 화가 나서 / 스트레스 받아서 / 기분 전환으로 / 충동구매 / 기타
   - 리스트는 추후 확정. 현재는 임시안.
      │
8. 장바구니에 추가
   - 누끼 이미지 + 가격 태그 형태로 장바구니에 배치
```

### 카테고리 임시안

| 카테고리 | 예시 품목 |
|---|---|
| 음식/음료 | 카페, 편의점, 배달음식 |
| 패션/뷰티 | 옷, 신발, 화장품 |
| 생활용품 | 청소용품, 식재료 |
| 문화/여가 | 영화, 공연, 책 |
| 교통 | 택시, 대중교통 |
| 건강 | 약, 운동 |
| 기타 | 분류 외 항목 |

## 7. 캘린더 화면

> 일부 UI 캡처본은 추후 제공 예정. 제공 후 부족한 요소를 추가 구현한다.

### 7.1 화면 구조

```
┌──────────────────────────────┐
│ [헤더]                        │
│  날짜 (예: 26.02)              │
├──────────────────────────────┤
│ [세그먼트 컨트롤]   [총 금액]   │
│   월간   │   주간              │
├──────────────────────────────┤
│                              │
│  [캘린더 영역]                 │
│  날짜 숫자 + 누끼 썸네일        │
│                              │
├──────────────────────────────┤
│  [하단 장바구니 카트]           │
│  해당 기간/날짜 물품 모아보기    │
│  (인터랙션: 흔들기 → 파도 모션)  │
├──────────────────────────────┤
│  [분석 영역]                   │
│  소비 통계 및 추가 정보          │
└──────────────────────────────┘
```

### 7.2 헤더

- 날짜 표시: `YY.MM` 형식 (예: 26.02)

### 7.3 세그먼트 컨트롤

- **월간**: 한 달 전체 캘린더 뷰
- **주간**: 해당 주 뷰

### 7.4 캘린더 영역

- 각 날짜 셀에 날짜 숫자 표시
- 날짜 숫자 위 또는 주변에 그날 소비한 아이템의 누끼 사진을 작게 배치
- 날짜 셀 클릭 시 → 홈 화면의 해당 날짜 장바구니 뷰로 이동
  - 홈 화면 상단에 캘린더로 돌아가는 버튼 노출
  - 버튼 클릭 시 캘린더로 복귀

### 7.5 하단 장바구니 카트

- 현재 선택된 기간 또는 날짜의 구매 물품을 장바구니 카트에 모두 담아서 보여줌
- **인터랙션**: 화면을 흔들거나 특정 터치 시 카트 내 아이템들이 파도치듯 움직이는 모션 고려 (세부 구현은 추후 확정)

### 7.6 분석 영역

| 항목 | 설명 |
|---|---|
| 가장 많이 소비한 품목 | 선택 기간 내 구매 빈도 Top N |
| 가장 비싼 물건 | 선택 기간 내 최고 금액 품목 |
| 소비 이유 통계 | 소비 이유별 분포 (예: "귀여워서 3건", "필요해서 5건") |
| 무지출일 수 | 선택 기간 내 무지출로 기록된 날 수 |

### 7.7 총 금액 표시

- 세그먼트 컨트롤 우측에 현재 보고 있는 기간의 총 소비 금액 표시

## 8. 친구 화면

### 8.1 핵심 방향

친구 기능은 비교·감시가 아니라, **소비 취향을 가볍게 공유하고 구경하는 경험**을 목표로 한다.  
친구의 장바구니를 피드가 아닌 '장바구니 구경' 방식으로 탐색할 수 있도록 설계한다.

### 8.2 주요 기능

| 기능 | 설명 |
|---|---|
| 친구 목록 | 연결된 친구 목록 표시. 앱 내 유저 검색으로 친구 추가 |
| 친구 그룹 | 친구를 그룹으로 묶어 관리. 그룹별로 공유 설정을 다르게 적용 가능 |
| 친구 장바구니 구경 | 친구의 장바구니를 날짜별로 탐색 |
| 이모지 반응 | 친구의 품목에 이모지 반응 남기기. v1에서는 댓글 제외 |

### 8.3 공유 범위 설정

- 공개 범위는 **그룹 단위**로 설정 (그룹별로 공개/비공개 선택)
- 품목별 개별 비공개 처리 가능
- 그룹 생성 시 **금액 노출 여부** 설정 가능
  - ON: 가격 태그 노출
  - OFF: 가격 태그 숨김

### 8.4 그룹 설정 항목

그룹 생성/수정 시 설정 가능한 항목:

- 그룹명
- 초대할 친구 선택
- 금액 노출 여부 (ON / OFF)
- 공개 범위 (공개 / 비공개)

### 8.5 v1 제외 항목

- 댓글 기능 (v2 이후)
- 친구 추천 알고리즘

## 9. 챌린지 시스템

### 9.1 목적

챌린지는 과도한 금융 관리 압박이 아니라, **가볍고 재미있는 재방문 동기**로 설계한다.

### 9.2 기본 구조

- 연속 N일 출석 또는 소비 기록 체크
- 연속 기록 달성 시 보상 지급

### 9.3 보상 예시

| 보상 유형 | 예시 |
|---|---|
| 장바구니 컬러 | 한정 파스텔 컬러, 메탈릭 컬러 등 |
| 꾸미기 요소 | 스티커, 스탬프, 장바구니 스킨 |
| 배지 | "7일 연속 기록왕", "무지출 챔피언" 등 |
| 기타 | 시각적 커스터마이징 요소 (추후 확정) |

### 9.4 챌린지 종류 (임시안)

| 챌린지 | 조건 | 보상 |
|---|---|---|
| 첫 기록 | 첫 번째 물건 담기 | 기본 배지 |
| 3일 연속 기록 | 3일 연속 기록 완료 | 컬러 1개 해금 |
| 7일 연속 기록 | 7일 연속 기록 완료 | 꾸미기 요소 1개 해금 |
| 무지출 챔피언 | 1주일 중 무지출 3일 이상 | 배지 |
| 30일 달성 | 한 달 중 20일 이상 기록 | 프리미엄 스킨 |

- 챌린지 목록 및 조건은 추후 확정. 현재는 임시안.

### 9.5 챌린지 노출 위치

- MY 탭 내 챌린지 현황 섹션에서 확인
- 보상 달성 시 홈 화면 또는 앱 내 알림으로 축하 피드백 제공

## 10. 디자인 가이드

> 세부 디자인 스펙은 추후 `design.md` 및 `theme.js` 파일로 별도 제공 예정.

### 10.1 톤 앤 무드

- **키치 (Kitschy)**: 밝고 유쾌한 컬러, 장바구니·영수증·가격 태그 등 물성이 느껴지는 UI 오브젝트
- **미니멀 (Minimal)**: 불필요한 정보 제거, 넓은 여백, 간결한 타이포그래피

### 10.2 컬러 시스템

| 역할 | 값 / 방향성 |
|---|---|
| Background | `#FFFFFF` (완전한 화이트) |
| Accent | `#111111` |
| Primary (장바구니 테마) | 날짜별로 사용자가 선택. 파스텔 계열 프리셋 제공 |
| Text | 다크 차콜 (고대비 유지) |

### 10.3 UI 컴포넌트 방향

**장바구니**
- 실제 사진 이미지 기반
- 소스: `iCloud Drive > 디자인나스 > 장바구니 > 장바구니 사진` 폴더 내 `장바구니_핑크`, `장바구니_밝은 그린` 등 활용
- 날짜별로 장바구니 테마 컬러 선택 가능

**영수증**
- 감열지 질감의 컬러 배경
- 각 영수증의 컬러는 해당 날짜의 장바구니 컬러에 대응
- 점선 구분선, 모노스페이스 폰트

**누끼 이미지**
- 그림자 효과 없음
- `#111111` 컬러로 **Offset Path 4px** 처리 → 스티커처럼 표현
- 각 누끼 이미지에 **1px 짧은 블랙 실선 + 작은 가격 태그** 부착
- 태그 클릭·드래그 시 실선이 늘어나는 인터랙션
- 장바구니 내 드래그로 위치 이동 가능

## 14. 오픈 이슈

1. **누끼 처리 품질** — 저화질 사진이나 복잡한 배경에서의 누끼 정확도 보장 방안
2. **장바구니 내 배치 알고리즘** — 누끼 이미지 간 겹침 방지 로직 (충돌 감지 + 재배치). 드래그 이동과의 조합 처리
3. **개인정보** — 소비 내역 이미지 포함 데이터의 보안·저장 정책
4. **소셜 온보딩** — 친구·그룹이 없는 신규 유저의 친구 탭 빈 화면 처리 방안
5. **누끼 API 비용** — 무료 플랜 한도 및 유료 전환 기준 설정 필요
6. **장바구니 사진 라이선스** — iCloud Drive 내 사진의 앱 사용 저작권 확인 필요
7. **소비 이유 리스트 확정** — 물건 추가 플로우의 소비 이유 항목 최종 확정 필요
8. **챌린지 보상 잠금 해제 조건** — 컬러·꾸미기 보상의 구체적인 달성 조건 설계 필요
